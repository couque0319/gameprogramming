# 🕹️ 게임프로그래밍 팀 프로젝트 제작과정

## 1. 프로젝트 개요

- 웹 기반 2D 탑뷰 탄막 슈팅 게임 개발
- HTML5 Canvas / JavaScript 중심 구조
- 스테이지 선택, 난이도(Easy/Hard), 웨이브 기반 적 스폰
- 플레이어 기체 선택(격납고), 상점 시스템, 아이템 업그레이드
- 보스/중간보스 이미지 및 패턴 구현

---

## 2. 주요 기능 구현 요청

### 스테이지 및 몬스터 구현

- 스테이지 1부터 탄막 슈팅 스타일의 적 패턴 구현
- Easy/Hard 모드의 몬스터 이동/탄속 차등 적용

**일반 몬스터 패턴:**
- 직선 발사
- 대각선 발사
- (추가 요청) 다양한 패턴 5종 설계

**보스 패턴:**
- 여러 단계(phase) 기반 설계
- 2개 또는 최대 5개의 패턴 구성
- 페이즈 전환 시 연출/무적/탄 클리어 처리

### 탄환 시스템

- 적 및 플레이어 탄환 이미지 적용
- 탄속 변경(예: 자동 공격 140으로 속도 조정)
- 탄환 충돌 판정 수정
- 탄환이 보이지 않던 문제 해결 및 생성 로직 수정

---

## 3. 코드 구조 및 리팩토링 요청

### 파일 구조 개편

- `game.js`가 너무 길어 분리 요청
- 이후 다시 통합 요청(game.js 하나로 합치기)
- `stage_manager.js` / `stage_defs.js` / `shop_event.js` 구조 제안·수정
- 몬스터 로직 모듈화 시도 → 다시 하나의 파일로 통합 필요

### 불필요한 기능 제거/수정

- "정지선 기능" 제거
- 플레이어 무적 처리 수정
- 튜토리얼 safe모드 비활성화
- HP 0일 때 모든 스테이지에서 게임오버 적용

### 코드 패치

- 중간보스 처치 후 이벤트(항모 이벤트) 미작동 → 수정 요청
- 탄환 충돌이 작동하지 않던 문제 해결 요청
- 스테이지 wave 정의가 game.js에서 보이지 않던 부분 확인 요청

---

## 4. 이미지 및 에셋 적용 관련 요청

### 몬스터 / 보스 이미지

- 일반 몬스터가 이미지 없이 도형으로 표시되는 문제 수정
- 보스 이미지 `assets/images/boss/` 내 중간보스/보스 파일 적용
- 보스 이미지 크기 조정(축소 요청)
- 동일 그림체로 추가 몬스터 이미지 제작 요청(AI 생성)
- 몬스터 도형 출력 방지 → 이미지 적용 강제

### 플레이어 HP 이미지 표시

- HP 숫자 → 하트 이미지로 교체
- 피격 시 `lost_HP.png`로 변경되도록 구현

---

## 5. 오디오 시스템 요청

### 효과음 적용

- 피격 시: `assets/audio/hit.mp3`
- 게임오버 시: `assets/audio/game_over.mp3`
- 게임오버 배너 등장 + 효과음 재생 처리

---

## 6. 아이템·상점 시스템

### 상점(shop_event.js) 관련 요청

- 기관포 업그레이드 2, 3단계 구현
- 미사일 언락 시 HUD에 표시(onBattle 상태에 따라 표시/숨김)
- 연사력 및 데미지 강화 기능 구현
- 데미지 강화 시 탄환 이미지 변경으로 시각적 효과

---

## 7. 스테이지와 난이도 시스템

### Easy 모드 조정

- 몬스터 탄속 1/2로 감소
- 스테이지별 스폰 및 패턴 커스터마이징 가능하도록 요청

### 기타

- 중간보스 → 항모 이벤트 미작동 문제 해결 요청
- 스테이지 잠금 해제 문제 수정

---

## 8. Phaser 버전 요청

- 기존 Canvas 기반 코드를 Phaser 방식으로 포팅 요청
- CDN 로드 방식 권장
- `stage_manager.js` Phaser 구조 완성본 요청

---

## 9. 오류·버그 해결 요청

### 주로 발생한 문제

- 몬스터 탄 생성/표시 안됨
- 보스 처치 후 화면 멈춤 → F12 에러 표시
- 이미지 경로 오류로 도형이 출력되는 문제
- 일정 패턴 제거 요청 (보스 패턴 제거 시 어느 부분 삭제해야 하는지)
- 코드 적용 후 플레이어·몬스터 충돌 처리 누락
- 웨이브가 작동하지 않거나 잘못된 타이밍 발생
